# 角色原型库使用指南

## 原型结构

```json
{
  "archetype_id": "ROLE-001",
  "category": "都市爱情",
  "role_name": "霸道总裁",
  "visual_traits": "棱角分明、剑眉星目、180+修长、偏分背头、薄唇冷峻",
  "outfit": "黑色三件套定制西装、白衬衫解扣、名表(百达翡丽)、锃亮皮鞋、袖扣",
  "common_scenes": "高层落地窗办公室、迈巴赫后座、高端酒会、私人别墅"
}
```

### 字段说明

| 字段 | 用途 |
|------|------|
| `archetype_id` | 唯一ID，格式 ROLE-NNN |
| `category` | 题材分类，用于 --category 过滤 |
| `role_name` | 角色名，直接写入 prompt 的「角色：」后面 |
| `visual_traits` | 视觉特征提示，用于推断性别/年龄段，**不直接写入 prompt** |
| `outfit` | 服装描述，写入 prompt 的「穿」后面 |
| `common_scenes` | 常见场景，仅供参考，不写入四视图 prompt（四视图是纯白背景） |

### visual_traits 的隐含作用

`visual_traits` 不直接出现在最终 prompt 中，但通过兼容规则影响 Face DNA 采样：
- 包含「父亲/母亲/大妈/长辈」等关键词 → 推断为老年角色 → 优先灰白发色、法令纹
- 包含「学生/校花」等关键词 → 推断为年轻角色 → 回避老化肤质
- 包含男性/女性关键词 → 影响发型池选择

## 扩展原型

新增角色时注意：
1. `archetype_id` 不能和已有重复
2. `category` 尽量使用已有分类；新建分类需在脚本 `get_theme_label()` 中添加映射
3. `visual_traits` 要包含性别和年龄暗示关键词，否则性别/年龄会随机分配
4. `outfit` 要具体到材质和颜色，避免模糊描述（如「时尚装扮」→ 改为「黑色西装三件套+白衬衫+皮鞋」）
5. 建议每个新题材至少 12 个原型，确保 30 组抽样有足够多样性
